可以，按你**当前代码与配置**，流程是这样的：

**1. 初始化阶段（确定抽取白名单与模式）**
1. `pdf_rag_pipeline.py` 里把 KG 质量参数定死为：`kg_extraction_mode="ppe"`、`kg_canonical_language="zh"`、`kg_relation_schema="fixed"`、`kg_merge_threshold=0.85`、`kg_llm_semantic_merge_enabled=False`。参考 [pdf_rag_pipeline.py](/media/disk2/wzh/RAG-Anything/pdf_rag_pipeline.py#L117)。
2. 同时传给 LightRAG：`entity_extract_max_gleaning=0`、`max_extract_input_tokens=5120`。参考 [pdf_rag_pipeline.py](/media/disk2/wzh/RAG-Anything/pdf_rag_pipeline.py#L203)。
3. `RAGAnything` 初始化 LightRAG 时，会把 `KGQualityManager` 里的实体/关系白名单写入 `addon_params`（用于抽取 prompt）。参考 [raganything.py](/media/disk2/wzh/RAG-Anything/raganything/raganything.py#L422)、[kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L394)。

**2. 文档处理入口**
1. 主入口是 `process_document_complete()`。参考 [processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L1777)。
2. 先做 PDF 解析，再做源头噪声过滤（header/page_number/页眉/二维码/路径等）。参考 [processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L1820)、[kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1319)。

**3. 文本链路：实体关系抽取**
1. 文本通过 `insert_text_content()` 进入 LightRAG，实际调用 `lightrag.ainsert(...)`。参考 [utils.py](/media/disk2/wzh/RAG-Anything/raganything/utils.py#L146)。
2. LightRAG 在 `extract_entities()` 用 `addon_params.entity_types/relationship_types` 构造抽取 prompt。参考 [operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L3266)。
3. 由于你设了 `entity_extract_max_gleaning=0`，不会跑第二轮 gleaning；`max_extract_input_tokens` 仅在 gleaning>0 时生效。参考 [operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L3358)。

**4. 多模态链路：实体关系抽取**
1. 多模态走 `_process_multimodal_content_batch_type_aware()`：先生成描述，再转 chunk，再调用 LightRAG `extract_entities()`。参考 [processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L893)、[processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L1486)。
2. 多模态抽取结果会先做一次 `manager.preprocess_chunk_results(...)`。参考 [processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L1510)。
3. 然后补 `belongs_to` 结构边，再做一次 preprocess。参考 [processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L1519)、[processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L1585)。

**5. 预合并治理（PPE）**
1. 你当前是 `ppe` 模式，走 `_preprocess_chunk_results_ppe()`：  
- Round1 规范化实体名/类型、加 `aliases`、过滤空多模态。  
- Round2 把“形态特征/危害症状/... ”这类节点投影为主实体 `attributes`。  
- Round3 只保留本体白名单关系。  
参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1697)。
2. 关系关键词会被映射到固定中文关系枚举（如 `致病/发生于/危害/防治...`），并做 domain-range 校验。参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1382)。
3. “别名”关系不会作为边保留，会写入节点别名属性。参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1440)、[kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1486)。

**6. 合并与落库（图 + 向量）**
1. `merge_nodes_and_edges()` 分三阶段：先实体、再关系、最后更新 `full_entities/full_relations`。参考 [operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L2875)。
2. 实体合并在 `_merge_nodes_then_upsert()`，会合并 `description/source_id/aliases/attributes`，写入图节点，再写入 `entities_vdb`。参考 [operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L2002)、[operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L2316)。
3. 实体向量内容现在包含 `实体名+类型+描述+aliases+attributes+模态`。参考 [operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L208)。
4. 关系合并在 `_merge_edges_then_upsert()`，写图边，再写 `relationships_vdb`。参考 [operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L2343)、[operate.py](/media/disk2/lhy/anaconda/envs/rag_anything/lib/python3.12/site-packages/lightrag/operate.py#L2812)。

**7. 后处理清洗（全图重写 + 去重）**
1. 文本和多模态都完成后，统一调用 `clean_graphml_file()` 重写 `graph_chunk_entity_relation.graphml`。参考 [processor.py](/media/disk2/wzh/RAG-Anything/raganything/processor.py#L134)、[kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1982)。
2. 这里会再做：节点/边规范化、属性投影、去掉无效边、PPE 节点类型裁剪、空多模态剔除。参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L2185)。
3. 去重现在是“向量主导 + LLM灰区仲裁”框架，但你当前配置 `kg_llm_semantic_merge_enabled=False`，所以只走向量严格合并。参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L773)、[config.py](/media/disk2/wzh/RAG-Anything/raganything/config.py#L216)。
4. 清洗结果会回写 GraphML，并返回统计。参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L2308)。

**8. 你当前这版的关键结论**
1. 抽取时：实体/关系类型已受本体白名单约束（通过 addon_params 注入）。  
2. 去重时：已默认关闭 LLM 仲裁，不会再卡在 LLM 合并上；仍有向量严格去重。  
3. 保存时：图（GraphML）+ 实体向量库 + 关系向量库都会更新。  
4. 备注：`kg_description_policy="multimodal_only"` 当前函数实现是 no-op（节点/边描述不被清空）。参考 [kg_quality.py](/media/disk2/wzh/RAG-Anything/raganything/kg_quality.py#L1233)。
