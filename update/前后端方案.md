## RAG-Anything 前端实现方案（Streamlit 一体化，结构化证据绑定）

### 1. 方案摘要
- 采用 `Streamlit + streamlit-echarts`，先做本机/内网版本，不引入鉴权。
- 上传仅支持 `PDF`（v1），每次上传创建独立知识库（独立 `working_dir` + `output_dir`）。
- 建库走异步任务，前端显示阶段进度（解析/文本入库/多模态处理/完成）。
- 问答返回结构化结果：`answer + citations + graph_focus`，支持“引用片段 + 多模态展开 + 图谱定位”。
- 图谱默认展示“回答相关子图”，提供“一键查看全图”。

### 2. 实现变更（按子系统）
- 新增 UI 入口与服务层（不改动核心 RAG 主流程）：`ui/app.py`、`ui/service.py`、`ui/job_manager.py`。
- `ui/app.py` 页面结构固定为三块：知识库管理（上传/选择/状态）、问答区（回答+引用）、图谱区（子图/全图切换）。
- `ui/job_manager.py` 负责异步建库：后台线程执行 `rag.process_document_complete()`，通过 `CallbackManager` 记录进度事件并供前端轮询。
- `ui/service.py` 封装核心接口：
  - `create_kb(pdf_file) -> {kb_id, job_id}`
  - `get_job(job_id) -> {status, stage, progress, events, error}`
  - `query(kb_id, question, mode="hybrid") -> QueryResponse`
  - `get_graph(kb_id, focus_chunk_ids=None, full=False) -> GraphPayload`
- 结构化证据绑定策略固定：
  - 回答文本：`rag.aquery(...)`
  - 检索证据：优先 `QueryParam(only_need_context=True)`，失败回退 `only_need_prompt=True`
  - 从上下文提取 `chunk_id`、`Image Path`、表格块；再从 `text_chunks/graphml` 补全 `file_path/page_idx/modality`
  - 基于 `graphml` 的 `source_id`（按 `<SEP>` 切分）反查命中节点/边，产出 `graph_focus`
- 多模态展示规则固定：
  - `image`：默认显示摘要，展开后 `st.image(image_path)`；文件不存在则显示原路径文本。
  - `table`：默认摘要，展开后优先展示 HTML 表格（`pandas.read_html`）；失败则原始文本回退。
  - `equation/generic`：展示引用片段与结构化字段，不强制渲染富媒体。

### 3. 接口与数据契约
- `QueryResponse` 固定字段：
  - `answer: str`
  - `citations: Citation[]`
  - `graph_focus: {node_ids: string[], edge_ids: string[], chunk_ids: string[]}`
  - `debug: {context_raw?: str}`（仅调试开关时返回）
- `Citation` 固定字段：
  - `chunk_id, file_path, page_idx, snippet, modality, source_type`
  - `asset_ref`（image path / table html / equation text）
- 图谱前端数据固定为 ECharts `nodes + links`，并额外携带 `highlight_node_ids/highlight_edge_ids` 做定位高亮。
- 目录约定固定：
  - `rag_storage_ui/{kb_id}`：知识库存储
  - `output_ui/{kb_id}`：解析产物
  - `uploads_ui/{kb_id}`：原始上传文件
  - `meta_ui/{kb_id}.json`：KB 元信息（文件名、创建时间、任务状态）

### 4. 测试与验收
- 单元测试：
  - 上下文解析：`chunk_id`/`Image Path` 抽取正确。
  - `source_id` 解析：`<SEP>` 多值映射到节点/边正确。
  - `graph_focus`：给定 `chunk_ids` 产出子图稳定。
- 集成测试：
  - 上传 `docs/test3_single.pdf` 完整跑通，任务状态可见且最终 `completed`。
  - 问答返回非空 `answer` 且至少 1 条 `citation`。
  - 命中图片/表格问题时可展开显示原图/完整表格。
  - 点击“定位图谱”后图谱高亮对应节点/边。
  - 切换不同 `kb_id` 时结果严格隔离。
- 异常测试：
  - 解析器缺失、文件损坏、任务失败时前端有明确错误信息与重试入口。
  - 证据提取失败时回退到“仅答案 + 空 citations”，UI 不崩溃。

### 5. 关键假设与默认值
- 运行环境具备 `lightrag-hku` 与解析依赖（当前仓库 requirements 为准）。
- v1 不做用户鉴权与多租户隔离，仅面向本机/内网单实例。
- v1 仅支持 PDF 上传；其他格式后续通过同一服务层扩展。
- 图谱默认先渲染子图，避免全图性能问题；全图需手动切换。
- 多模态默认“摘要折叠 + 手动展开”，不做自动全量展开。
